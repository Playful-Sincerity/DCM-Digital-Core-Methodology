# Wisdom's Speech — Digital Core Software Paradigm (2026-05-02)

## Core Vision Monologue

> There's a new paradigm that we're also pioneering now where you have the digital core within custom software where you have an agentic coding editor within your software and it's not SaaS anymore. It's now digital core software is the new paradigm where people can exist within an agentic system where essentially software they own, it's in their own digital core and they can edit it and then they can get updates from different people. People can now consult for setting up these custom systems that they can then have much more agency over. All of the data is within their own system and they can operate from it. Things like the CRB is a clear example of that and it's really powerful and important.

## Build/Use Collapse Monologue

> It's pretty crazy rather than being able to have an agentic system while you're building a software and then you have to deploy the software and use it. You're literally able to use an agentic system inside the software while you're using it in order to change it. That's a new paradigm people have been talking about for a little while now. And it looks like we're kind of pioneering it a little bit here.

## Business Model Monologue

> And it's this new world where you don't sell a subscription to a software — you sell consulting to build a custom software that then a user can actually use and edit and have agency over and everything. It's really exciting and that's why Happy Human Agents — right, the way we're building this — is really powerfully positioned for this new age.

## Pattern Observations

- The paradigm has three claims layered together: (1) the agentic editor lives inside the running software, not separate from it; (2) the user owns the software and can keep modifying it; (3) the business model is consulting-to-build-custom-software, not subscription-to-shared-software.
- Wisdom hedged correctly on novelty: *"people have been talking about for a little while now"* — we're early instantiators, not sole inventors. Preserve that hedge in the paper.
- HHA is positioned as the canonical demonstration of the paradigm, not as one case study among many. CRB named as the clearest example.
- The implication for the DCM paper's audience scope is direct: this isn't just for academic researchers — it's for anyone considering building custom software with an embedded agentic editor that they or their clients can keep editing.
