## Late Afternoon — [Discovery]

**What:** Inline base64 embedding in `use_figma` calls has a character-mangling failure mode. 4 of 6 calculus formula PNGs didn't render when embedded inline — the NSA column worked, the classical and IVNA columns failed.
**Why:** Long base64 strings in the code field apparently have certain character sequences that get corrupted in transit between the tool and the Figma sandbox.
**Means:** Workaround: push the PNGs to the public GitHub repo and use `figma.createImageAsync(url)` to fetch them. This avoids the embedding size limit entirely and is more reliable. The IVNA QR code and the main formula were small enough to survive; the calculus images were on the edge.
**How to apply:** For future Figma builds with images, prefer URL fetch over base64 embedding whenever possible.
